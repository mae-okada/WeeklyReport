# About the Weekly Report

This document describes the data included in the weekly reports generated by the WeeklyReportApp and the logic used to filter and process the data.

## Overview

The application generates two types of reports:
1. **Stage Change Report** (`{date}_ステージ変更リスト.txt`)
2. **Weekly Sales Report** (`{date}_週刊レポート.txt`)

Both reports are based on data from two Excel files: the most recent and the previous one in the `data/` folder. The reports are generated weekly and include information about deals (projects) in various sales stages.

## Data Source

- **Input Files**: Two Excel files from the `data/` folder, selected as the latest two by filename.
- **Data Structure**: Each Excel file contains deal information with columns including ID, Company, Name (project), Stage, Size, Owner Fullname, and various stage-specific fields.
- **Processing**: Data is loaded starting from row 3 (header row 2), with unnamed columns removed.

## Stage Mapping

Deals are grouped by Japanese stage labels using the following mapping:

stage_map = {

## Stage Change Report

### Purpose
This report shows deals that have changed stages or sizes between the two data files, plus new deals and upcoming renewals.

### Filtering Logic

1. **Stage Changes**: Deals where the stage changed between old and new files, excluding changes from stage "5. Sales (Invoice)".
2. **New Deals**: Deals present in the new file but not in the old file.
3. **Size Changes**: Deals where the "Size" field changed between files.
4. **Upcoming Renewals**: Deals in stage "1-2. Potential (Renewal)" with effective dates in the current or next month.

### Processing Steps
1. Detect stage changes and new deals.
2. Detect size changes.
3. Merge size change data with new file data.
4. Combine all detected changes, removing duplicates by ID.
5. Add filtered renewal deals.
6. Sort by size in descending order.

### Output Format
- Grouped by Japanese stage labels.
- Each deal shows: company name, size (with change indicator if applicable), project name (translated to Japanese), owner name, and stage change information.
- Sorted by deal size within each group.

## Weekly Sales Report

### Purpose
This report shows sales-owned deals and S/O stage deals that have changed, providing a weekly overview of active sales pipeline.

### Filtering Logic

1. **Changed S/O Deals**: Deals in "4. S/O" stage that had stage changes.
2. **Sales-Owned Deals**: Deals owned by specific sales team members:
   - MGTI okada
   - MGTI Barri
   - MGTI Luky
   - Katsuhiro Motono
3. **Sales-Owned Renewals**: Renewal deals (stage "1-2") owned by the above sales team, with effective dates in current/next month.
4. **Exclusion**: Deals in "1-2" stage are included only if they are renewals with upcoming effective dates.

### Processing Steps
1. Detect all stage changes.
2. Extract "4. S/O" stage deals from changes.
3. Remove "4. S/O" stage deals from the new data file.
4. Combine the extracted S/O changes with the remaining new data.
5. Filter for sales-owned deals using the criteria above.
6. Remove duplicates by ID.
7. Sort by size in descending order.

### Output Format
- Grouped by Japanese stage labels.
- Each deal shows: company name, size (with change indicator if applicable), project name (translated to Japanese), and stage change information.
- No owner names are included.
- Sorted by deal size within each group.

## Row Formatting Details

Each deal row in the reports includes:

- **Company Name**: Cleaned company name.
- **Size**: Deal size in "百万円" format, with change indicator if size changed (e.g., "100 (+50)" or "100 (-25)").
- **Project Name**: Translated to Japanese using the translator service.
- **Stage-Specific Additions**:
  - Stage 3: Goal grade and "注文書待ち".
  - Stage 4: "請求書発行".
  - Stage 5: "請求書送付済み".
- **Owner Name**: Only in Stage Change Report.
- **Stage Change Info**: Shows previous → current stage, or "【新規案件 → stage】" for new renewals, or "【初】" for completely new deals.

## Additional Processing

- **Translation**: Project names are translated to Japanese using the TranslatorService.
- **Cleaning**: Company and project names are cleaned of special characters.
- **Size Conversion**: Sizes are converted to "百万円" format.
- **Date Handling**: Renewal effective dates are parsed and filtered by month.

## Output Files

Reports are saved as UTF-8 encoded text files in the `output/` folder with today's date in the filename.
